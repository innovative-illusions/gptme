Contributors#

Thanks to everyone who contributed to this release:

@erikbjare

Changelog#

Changes since v0.16.0:

📦 gptme#

✨ Features (1)#

  • feat: add support for XML-formatted tool calls (#121) (d0b070fc)

🐛 Fixes (23)#

Click to expand

  • fix: changed warning log to debug (0c4c1869)

  • fix: minor eval refactor (improved type names), clarified python tool instructions (e0c79a41)

  • fix: disable tqdm in tests (77ef4fa1)

  • fix: reduced number of decimals in eval output (b216179b)

  • fix: removed spammy message when not in a git repo (be3d0232)

  • fix: futher reliability improvements to evals (622e5744)

  • fix: added tqdm progress bar to eval (cd33e06d)

  • fix: nitpick (11f1e7ee)

  • fix: improved typing in gptme.evals.run (6d00be7a)

  • fix: more fixes, speed up list_chats and search_chats by lazily searching chronologically (8c3cb778)

  • fix: more fixes and store eval case results in result directory (1e23ecbb)

  • fix: comment out warning for unknown codeblock types, add lru_cache to frequently called get_tool_for_langtag, added wip llm_openai.list_models (aff213f6)

  • fix: refactored evals, fixed leaked semaphore warnings, read logs and naively compute tokens from output (3bbd88b4)

  • fix: improved system prompt, added system prompt to docs (#123) (62220b1a)

  • fix: refactored and improved evals (#122) (e4eb81ca)

  • fix: improved eval stream capturing logic (eecabac8)

  • fix: improved evals output capturing, don’t capture by default if a single test is run, and write streams to results directory (8e9ad4c1)

  • fix: process eval run futures in the order they are finished, instead of waiting in order (5601fae5)

  • fix: remove the missing datascience packages warning at python tool init (09f115f2)

  • fix: add ignorelist for certain known non-executable codeblock langs (7b82b033)

  • fix: lowered logging level for some spammy messages (9a729af0)

  • fix: disable placeholder-aware patching if placeholders in original file, improve error message if file not found (1a1e9fb6)

  • fix: dont ask for version in bump_version script if already on tag and pyproject updated (68aae660)
